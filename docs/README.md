# World Cup 2026 Player Guide

Last reviewed: 2026-06-27

This repository contains a static interactive guide for browsing and comparing World Cup 2026 player-card data. The app is plain HTML, CSS, and browser JavaScript served by nginx in Cloud Run. There is no package manager, bundler, or framework build step.

## Production

- Production domain: https://wc26.parthshah.dev
- Backing Cloud Run service URL: https://world-cup-2026-guide-qhtwqt3clq-uc.a.run.app
- GCP project: `parth-projects-500703`
- Region: `us-central1`
- Cloud Run service: `world-cup-2026-guide`
- Artifact Registry repository: `world-cup-apps`

Use the custom production domain for user-facing checks. Treat the Cloud Run URL as the backing service address.

## App Surfaces

- `index.html`: country-card home page and legacy query-route redirects.
- `team.html`: single-country page with team metrics and sortable player cards.
- `players.html`: global Top Players page with search, filters, sorting, and 24-card pagination.
- `compare.html`: side-by-side country comparison with each selected team's full player list.

The global Top Players page intentionally paginates player cards to keep smaller/mobile browsers from rendering the whole player dataset at once.

## Code Map

- `js/pages/`: page-specific controllers.
- `js/components/`: reusable render helpers for cards, lists, metrics, and the topbar.
- `js/shared/`: data loading, URL utilities, theme, and stats helpers.
- `styles/app.css`: all app styling and responsive behavior.
- `scripts/build-data.js`: converts the source CSV and flag manifest into browser-ready JSON.
- `scripts/deploy-gcloud.sh`: builds and deploys the nginx container to Cloud Run.
- `nginx.conf`: Cloud Run nginx config and cache headers.

## Data Files

- Source CSV: `world_cup_player_tables_collected.csv`
- Generated global payload: `data/players-lite.json`
- Generated team summaries: `data/teams-summary.json`
- Generated per-team payloads: `data/teams/*.json`
- Flag manifest: `assets/flags/manifest.json`

After changing the source CSV or flag manifest, regenerate the JSON files:

```bash
node scripts/build-data.js
```

See [data-methodology.md](data-methodology.md) for dataset scope, field definitions, and source hierarchy.

## Local Development

Serve the repository root:

```bash
python3 -m http.server 8765
```

Open http://127.0.0.1:8765/ in a browser.

Recommended smoke checks:

- Home page loads country cards.
- `players.html` renders 24 player cards on the first page and the Next button advances the range.
- `team.html?team=france` loads a team page.
- `compare.html` allows selecting two countries and shows each team's full player list.

## Deployment

Deploy the current checkout to Cloud Run:

```bash
./scripts/deploy-gcloud.sh
```

Optional overrides:

```bash
REGION=us-central1 SERVICE_NAME=world-cup-2026-guide ./scripts/deploy-gcloud.sh
```

The deploy script enables required GCP APIs, creates the Artifact Registry Docker repository if missing, submits a Cloud Build, deploys Cloud Run, and prints the backing service URL.

After deployment, verify the custom production domain:

```bash
open https://wc26.parthshah.dev/players.html
```
